Prism Element credentials required

Prism Central credentials don't authenticate to . Enter local PE credentials. They are validated against PE and held only in the NRCC server's memory for this session. They are never written to browser storage and never persisted to disk. Use Forget PE credentials on the toolbar to clear them, or restart the NRCC server.

Settings
Appearance
Session

When idle for the selected interval, NRCC clears your in-memory credentials, closes any open consoles, and returns to the sign-in screen.

Logging

Activity is appended to a weekly-rotated log file on the NRCC server (one file per ISO week). Each entry records the timestamp, your username, and the action.

Logging is disabled at the server level (set NRCC_LOGGING=true to enable it).

Beta features

Reveals UI for features that are still under development. Beta features may change or be removed without notice.

Individual beta features
Inventory

Speeds up VM enumeration by skipping hidden/system VM discovery and CVM-specific lookup probes. Regular VM name and power-state inventory still loads.

Server (affects all users)

Activity-log files older than this many days are deleted automatically by a sweep that runs every 6 hours. 0 = keep forever. The default is 30 days. This setting is shared across the whole NRCC deployment.

Activity logging is off, so retention has no effect until NRCC_LOGGING=true is set.
